this is my first time trying to code a webpage inside of a docker container running on a raspberry pi

How was this achieved?

this was achieved by the following

  1. first setting up the raspberry pi
  2. loading a docker container
  3. using linuxserver.io's vscode server image

The rough start

starting the container and getting everything working was easy, but I kept running into a problem with git, before realizing after a few hours that i hadn't set git up properly.

I've had prior experience in making a docker image before, using a raspberry pi zero 2 running pihole, but haven't really touched it much besides pulling new images and recreating them.

The Journey so far

  1. imaged a RasPiOS Lite 64 bit onto pi 4 8gb model
  2. 52 pi wiki for EP-0152 board oled setup and fan control
    • also looked at another youtuber's video on having the fan speed up and slow down, but created too much of a buzzing noise, so i just opted for the regular in-built fan controls via raspi-config
  3. followed instructions from TechCraft youtube to enable usb-c OTG
  4. created bash aliases
    • alias stress="stress-ng --cpu 4 --cpu-method fft"
    • alias temp="watch -n 1 vcgencmd measure_temp; vcgencmd measure_volts; vcgencmd get_throttled"
    • alias shutdown="sudo shutdown -h now"
  5. ssh keygen copy to pi without having to constantly enter password
  6. install docker, create docker-compose.yml file, set up code-server using linuxserver.io's docker image
  7. created git and uploaded repo
  8. started configuring an efficient VNC without using raspi-config's built in realVNC enable
    • used a script from https://github.com/gitbls/RPiVNCHowTo
    • one problem I ran into was that I couldn't find the wifi icon(still cant) and at some point the wlan0 interface stopped working completely
      • I had to end up re-typing ssid and password using raspi-config for it to work again, but im not sure why it was like that as the information was already inside of wpa_supplicant
    • KDE-Plasma might be too heavy for the pi, so I'm going to try to trim it back down to XFCE or something
  9. tested deploy keys in github on my pi and ipad to see how pushing code through git using the keys works
    • initially I created a separate key inside of the container thinking that it would work but it still kept asking for my login credentials, until i changed the keys to my ipad's key
